# Data Engineering Authority

## Database: PostgreSQL 16

### Connection
- Host: `localhost`
- Port: `5433` (mapped from container 5432)
- Database: `scripturequest`
- User: `sq` / Password: `sq` (dev only)
- ORM: Drizzle ORM with postgres.js driver

### Schema Overview

```
users ──┬── userProgress ──── lessons ──── lessonSteps
        │                       │
        ├── quizAttempts ──── quizzes ──── questions
        │
        ├── userBadges ────── badges
        │
        ├── streaks
        │
        └── contentUploads

lessons ──── series
lessons ──── lessonReviews
```

### Key Relationships
- `users` 1:N `userProgress` (tracks lesson completion)
- `lessons` 1:N `lessonSteps` (ordered content steps)
- `lessons` 1:1 `quizzes` (each lesson has one quiz)
- `quizzes` 1:N `questions` (quiz questions)
- `users` 1:N `quizAttempts` (quiz attempt history)
- `series` 1:N `lessons` (content grouping)
- `users` N:N `badges` (via `userBadges`)

### Enums
- `ageGroupEnum`: `little-explorers`, `junior-scholars`, `young-disciples`
- `userRoleEnum`: `student`, `parent`, `teacher`, `admin`
- `lessonStatusEnum`: `draft`, `review`, `published`, `archived`
- `questionTypeEnum`: `multiple_choice`, `true_false`, `fill_blank`

## Migrations

- Generate: `npm run db:generate`
- Apply: `npx drizzle-kit push:pg`
- Never edit generated migration files
- Test migrations against a copy of prod data before applying

## Data Integrity Rules

- All IDs are UUIDs (generated by PostgreSQL `gen_random_uuid()`)
- `createdAt` / `updatedAt` on all tables
- Foreign keys with `CASCADE` on delete where appropriate
- No orphaned records — clean up related data on delete
- Unique constraints: user email, badge name, series title

## Query Patterns

- Use Drizzle query builder, never raw SQL
- Named imports from schema
- Select only needed columns for list views
- Use transactions for multi-table writes
- Paginate all list endpoints (limit + offset or cursor)
